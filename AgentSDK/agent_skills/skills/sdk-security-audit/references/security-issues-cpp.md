# 安全审计问题类型

本文档列出 SDK 安全审计中需要关注的问题类型和危险点，参考华为昇腾社区 C++ 语言安全编程指导。

---

## 1. 安全编码

### 1.1 总体原则

| 规则编号 | 规则名称 | 风险等级 | 说明 |
|---------|---------|---------|------|
| 1.1.1 | 保证静态类型安全 | 高 | 约束联合体、类型转换、缩窄转换、类型退化、范围错误、void* 类型指针等特性的使用 |
| 1.1.2 | 保证内存安全 | 极高 | 防止内存越界访问、释放后继续访问、解引用空指针、内存未初始化、局部变量地址逃逸、资源未释放 |
| 1.1.3 | 禁止使用编译器"未定义行为" | 高 | 遵循ISO C++标准，禁止使用标准中未定义的行为，谨慎使用编译器扩展特性 |

### 1.2 类

| 规则编号 | 规则名称 | 风险等级 | 说明 |
|---------|---------|---------|------|
| 1.2.1 | 类的成员变量必须显式初始化 | 高 | 避免对象状态处于不确定性，编译器生成的构造函数不会对成员变量进行初始化 |

**例外情况：**
- 如果类的成员变量具有默认构造函数，可以不需要显式初始化
- 如果类的设计目的就是将成员变量作为未初始内存块使用，可以不初始化

### 1.3 表达式与语句

| 规则编号 | 规则名称 | 风险等级 | 说明 |
|---------|---------|---------|------|
| 1.3.1 | 确保有符号整数运算不溢出 | 极高 | 有符号整数溢出是未定义行为，需在指针运算、数组索引、内存拷贝长度、内存分配参数、循环判断条件等场景校验 |
| 1.3.2 | 确保无符号整数运算不回绕 | 极高 | 无符号整数回绕可能导致缓冲区溢出、内存分配不足等问题 |
| 1.3.3 | 确保除法和余数运算不会导致除零错误 | 极高 | 整数除法和取余运算的第二个操作数为0会导致未定义行为 |
| 1.3.4 | &&和\|\|操作符的右侧操作数不要包含副作用 | 中 | 右操作数是否被求值取决于左操作数，副作用可能不会发生 |
| 1.3.5 | 循环必须安全退出 | 中 | 重复提供服务的逻辑循环应当设计退出机制，并正确释放资源 |

#### 规则 1.3.1 详细说明 - 有符号整数溢出检查

**加法溢出检查：**
```cpp
if (((numA > 0) && (numB > (INT_MAX - numA))) ||
    ((numA < 0) && (numB < (INT_MIN - numA)))) {
    // 错误处理
}
```

**减法溢出检查：**
```cpp
// 使用 size_t 类型表示数据长度，并校验外部数据长度是否在合法范围内
if (skipLen >= totalLen || totalLen > contentSize) {
    // 错误处理
}
```

**乘法溢出检查：**
```cpp
if (opt > (INT_MAX / (60 * HZ))) {
    // 错误处理
}
```

**除法溢出检查：**
```cpp
// 检查除数为0及除法溢出错误（INT_MIN / -1 会溢出）
if ((numB == 0) || ((numA == INT_MIN) && (numB == -1))) {
    // 错误处理
}
```

**一元减溢出检查：**
```cpp
if (numA == INT_MIN) {
    // 错误处理
}
```

#### 规则 1.3.2 详细说明 - 无符号整数回绕检查

**加法回绕检查：**
```cpp
// 使用减法替代加法进行比较
if (pktLen > totalLen - readLen) {
    // 错误处理
}
```

**减法回绕检查：**
```cpp
// 调整减法运算位置，确保编译期间减法表达式的值不翻转
if (len > SCTP_SIZE_MAX - sizeof(SctpAuthBytes)) {
    // 错误处理
}
```

**乘法回绕检查：**
```cpp
if (width == 0 || height == 0) {
    // 错误处理
}
if (width > SIZE_MAX / height) {
    // 错误处理
}
```

### 1.4 资源管理

| 规则编号 | 规则名称 | 风险等级 | 说明 |
|---------|---------|---------|------|
| 1.4.1 | 外部数据作为数组索引或者内存操作长度时，需要校验其合法性 | 极高 | 防止越界访问和缓冲区溢出 |
| 1.4.2 | 内存申请前，必须对申请内存大小进行合法性校验 | 高 | 防止整数溢出导致分配不足 |
| 1.4.3 | 禁止将局部变量的地址传递到其作用域外 | 极高 | 局部变量销毁后，指向它的指针成为悬挂指针 |
| 1.4.4 | 自定义new/delete操作符需要配对定义，且行为与被替换的操作符一致 | 高 | 确保内存管理的一致性 |
| 1.4.5 | 使用恰当的方式处理new操作符的内存分配错误 | 高 | 正确处理内存分配失败情况 |

### 1.5 错误处理

| 规则编号 | 规则名称 | 风险等级 | 说明 |
|---------|---------|---------|------|
| 1.5.1 | 禁止从析构函数中抛出异常 | 极高 | 析构函数抛出异常会直接导致 std::terminate |

### 1.6 标准库

| 规则编号 | 规则名称 | 风险等级 | 说明 |
|---------|---------|---------|------|
| 1.6.1 | 禁止从空指针创建std::string | 极高 | 会导致未定义行为 |
| 1.6.2 | 确保用于字符串操作的缓冲区有足够的空间容纳字符数据和结束符，并且字符串以null结束符结束 | 极高 | 防止缓冲区溢出和字符串处理错误 |
| 1.6.3 | 外部数据用于容器索引或迭代器时必须确保在有效范围内 | 高 | 防止容器越界访问 |
| 1.6.4 | 调用格式化输入/输出函数时，使用有效的格式字符串 | 高 | 防止格式化字符串漏洞 |
| 1.6.5 | 调用格式化输入/输出函数时，禁止format参数受外部数据控制 | 极高 | 防止格式化字符串漏洞 |
| 1.6.6 | 禁用程序与线程的退出函数和atexit函数 | 高 | 避免资源清理问题 |
| 1.6.7 | 禁止调用kill、TerminateProcess函数直接终止其他进程 | 高 | 会导致其他进程的资源得不到清理 |
| 1.6.8 | 禁止在信号处理例程中调用非异步安全函数 | 高 | 可能导致死锁或数据竞争 |

### 1.7 函数使用

| 规则编号 | 规则名称 | 风险等级 | 说明 |
|---------|---------|---------|------|
| 1.7.1 | 禁止直接使用外部数据拼接SQL命令 | 极高 | 防止SQL注入攻击 |
| 1.7.2 | 禁止使用非安全退出函数 | 高 | 以下函数禁止使用：kill、TerminateProcess、pthread_exit、ExitThread、exit、ExitProcess、abort |

### 1.8 内存

| 规则编号 | 规则名称 | 风险等级 | 说明 |
|---------|---------|---------|------|
| 1.8.1 | 严禁使用string类存储敏感信息 | 高 | string类无法保证敏感信息被清零，敏感信息可能散落到内存各处 |
| 1.8.2 | 内存中的敏感信息使用完毕后立即清0 | 高 | 避免敏感信息被攻击者获取 |

**正确做法示例：**
```cpp
// 使用 char 或 unsigned char 保存敏感信息
char password[MAX_PASSWORD] = {0};
GetPassword(password, sizeof(password));
VerifyPassword(password);
// 使用完毕后清零
memset_s(password, 0, sizeof(password));
```

### 1.9 文件

| 规则编号 | 规则名称 | 风险等级 | 说明 |
|---------|---------|---------|------|
| 1.9.1 | 外部文件路径使用前必须进行规范化并校验 | 极高 | 防止路径遍历攻击，验证路径中不包含 `..`、符号链接等 |

### 1.10 其他

| 规则编号 | 规则名称 | 风险等级 | 说明 |
|---------|---------|---------|------|
| 1.10.1 | 禁用rand函数产生用于安全用途的伪随机数 | 高 | rand() 生成的是伪随机数，应使用 /dev/random 生成随机数 |
| 1.10.2 | 禁止在发布版本中输出对象或函数的地址 | 中 | 可能泄露敏感的内存布局信息 |

## 危险点清单

### 内存与字符串函数

| 函数 | 风险等级 | 说明 |
|------|---------|------|
| `memcpy` | 高 | 需确保目标缓冲区大小足够 |
| `memmove` | 中 | 相对安全，但仍需检查边界 |
| `memset` | 中 | 需确保目标缓冲区大小 |
| `strcpy` | 极高 | 极易导致缓冲区溢出 |
| `strncpy` | 高 | 可能不终止字符串 |
| `strcat` | 极高 | 极易导致缓冲区溢出 |
| `strncat` | 高 | 需确保目标缓冲区大小 |
| `sprintf` | 极高 | 极易导致缓冲区溢出 |
| `snprintf` | 中 | 相对安全，但仍需检查返回值 |
| `vsprintf` | 极高 | 极易导致缓冲区溢出 |
| `vsnprintf` | 中 | 相对安全 |
| `sscanf` | 高 | 需确保输入格式正确 |
| `strtok` | 中 | 非线程安全 |

### 分配与释放函数

| 函数 | 风险等级 | 说明 |
|------|---------|------|
| `malloc` | 中 | 需检查返回值 |
| `calloc` | 中 | 需检查返回值 |
| `realloc` | 高 | 可能导致内存泄漏或 UAF |
| `free` | 高 | 需确保指针有效且未重复释放 |
| `new` | 中 | 可能抛出异常 |
| `new[]` | 中 | 需与 `delete[]` 配对 |
| `delete` | 高 | 需确保指针有效 |
| `delete[]` | 高 | 需与 `new[]` 配对 |

### 文件与系统函数

| 函数 | 风险等级 | 说明 |
|------|---------|------|
| `open` | 中 | 需检查返回值 |
| `fopen` | 中 | 需检查返回值 |
| `freopen` | 高 | 可能导致资源泄漏 |
| `unlink` | 高 | 可能导致数据丢失 |
| `remove` | 高 | 可能导致数据丢失 |
| `rename` | 中 | 需检查返回值 |
| `mktemp` | 极高 | 不安全，存在竞争条件 |
| `system` | 极高 | 命令注入风险 |
| `popen` | 极高 | 命令注入风险 |
| `exec*` | 高 | 参数注入风险 |

### 并发相关

| 操作 | 风险等级 | 说明 |
|------|---------|------|
| mutex lock/unlock | 高 | 需确保成对出现，避免死锁 |
| rwlock | 高 | 需确保读写锁正确使用 |
| spinlock | 高 | 需确保锁释放，避免死锁 |
| atomic state | 中 | 需确保原子操作正确 |
| condition variable | 高 | 需确保正确等待和通知 |
| callback register/unregister | 高 | 需确保生命周期正确 |
| thread start/join/detach | 高 | 需确保线程正确管理 |

### 禁止使用的函数

| 函数 | 风险等级 | 说明 |
|------|---------|------|
| `kill` | 极高 | 强行终止其他进程，资源得不到清理 |
| `TerminateProcess` | 极高 | 强行终止其他进程，资源得不到清理 |
| `pthread_exit` | 极高 | 线程内主动终止自身线程 |
| `ExitThread` | 极高 | 线程内主动终止自身线程 |
| `exit` | 极高 | main函数以外禁止调用 |
| `ExitProcess` | 极高 | main函数以外禁止调用 |
| `abort` | 极高 | 程序立即退出，资源得不到清理 |
| `rand` | 高 | 伪随机数，不应用于安全用途 |

## 审计检查清单

### 内存安全检查
- [ ] 是否存在内存越界访问？
- [ ] 是否存在释放后继续访问内存？
- [ ] 是否存在解引用空指针？
- [ ] 内存是否正确初始化？
- [ ] 是否存在局部变量地址逃逸？
- [ ] 申请的内存或资源是否及时释放？

### 整数安全检查
- [ ] 有符号整数运算是否防止溢出？
- [ ] 无符号整数运算是否防止回绕？
- [ ] 除法和取余运算是否防止除零错误？

### 资源管理检查
- [ ] 外部数据作为数组索引或内存操作长度时是否校验？
- [ ] 内存申请前是否对申请大小进行合法性校验？
- [ ] 是否禁止将局部变量地址传递到作用域外？

### 标准库检查
- [ ] 是否禁止从空指针创建 std::string？
- [ ] 字符串操作缓冲区是否有足够空间？
- [ ] 外部数据用于容器索引时是否确保有效？
- [ ] 格式化函数的 format 参数是否受外部数据控制？

### 敏感信息检查
- [ ] 是否使用 string 类存储敏感信息？
- [ ] 敏感信息使用完毕后是否立即清零？

### 文件安全检查
- [ ] 外部文件路径是否进行规范化并校验？
