# Python 安全编码规范

本文档列出 Python SDK 安全审计中需要关注的问题类型和危险点，参考华为昇腾社区 Python 语言安全编程指导。

---

## 1. 数据校验

### 1.1 校验跨信任边界传递的不可信数据

| 规则编号 | 规则名称 | 风险等级 | 说明 |
|---------|---------|---------|------|
| 1.1 | 校验跨信任边界传递的不可信数据 | 高 | 程序接收来自用户、网络连接或其他来源的不可信数据时，必须进行校验，且数据校验必须在信任边界以内进行 |

**校验策略（优先使用白名单）：**

| 策略 | 说明 | 示例 |
|------|------|------|
| 白名单校验 | 检查数据是否属于严格约束的合法数据集合 | `if not re.match("^[0-9A-Za-z_]+$", name): raise IllegalArgumentException` |
| 黑名单校验 | 拒绝已知坏的数据（较弱） | 剔除或转换某些危险字符 |
| 白名单净化 | 删除/编码/替换非合法字符 | `urllib.quote(input)` |
| 黑名单净化 | 剔除或转换某些字符（较弱） | `input.replace("'", "&rsquo;")` |

### 1.2 禁止直接使用不可信数据来拼接SQL语句

| 规则编号 | 规则名称 | 风险等级 | 说明 |
|---------|---------|---------|------|
| 1.2 | 禁止直接使用不可信数据来拼接SQL语句 | 极高 | 使用参数化查询，或对不可信数据进行校验 |

**正确做法：**
```python
args = (id, type)
cur.execute('select id, type, name from xl_bugs where id = %s and type = %s', args)
```

**错误做法：**
```python
sql = "select id, type, name from xl_bugs where id = %s and type = %s" % (id, type)
cur.execute(sql)
```

### 1.3 验证路径之前应该先将其标准化

| 规则编号 | 规则名称 | 风险等级 | 说明 |
|---------|---------|---------|------|
| 1.3 | 验证路径之前应该先将其标准化 | 极高 | 使用 `os.path.realpath()` 解析符号链接、快捷方式等，防止路径遍历攻击 |

**正确做法：**
```python
turepath = os.path.realpath(user_path)
if turepath.startswith('/home/python/'):
    open(turepath)
```

**错误做法：**
```python
print os.path.abspath("pwck")
```

### 1.4 禁止使用eval和exec执行不可信代码

| 规则编号 | 规则名称 | 风险等级 | 说明 |
|---------|---------|---------|------|
| 1.4 | 禁止使用eval和exec执行不可信代码 | 极高 | eval/exec 可执行任意 Python 代码，导致代码注入。如果必须使用，需对代码名称空间中的变量做黑名单校验 |

**错误做法：**
```python
guess = eval(raw_input('Your guess> '))
```

**正确做法：**
```python
guess = raw_input('Your guess> ')
if guess != str(num):
    print "Nope,", guess, "is wrong."
```

### 1.5 禁止使用input函数

| 规则编号 | 规则名称 | 风险等级 | 说明 |
|---------|---------|---------|------|
| 1.5 | 禁止使用input函数 | 极高 | Python2 中 `input()` 等同于 `eval(raw_input())`，可导致用户非法访问程序名称空间变量，应使用 `raw_input()` |

**错误做法：**
```python
Secret = "42"
value = input("Answer to everything is ? ")
```

**正确做法：**
```python
Secret = "42"
value = raw_input("Answer to everything is ? ")
```

### 1.6 禁止调用OS命令解析器执行命令或运行程序，防止命令注入

| 规则编号 | 规则名称 | 风险等级 | 说明 |
|---------|---------|---------|------|
| 1.6 | 禁止调用OS命令解析器执行命令或运行程序，防止命令注入 | 极高 | 使用未经校验的不可信输入作为系统命令的参数，可能导致命令注入漏洞 |

**注意事项：**
1. 命令执行的字符串不要拼接输入的参数，如果必须拼接时，要对输入参数进行白名单过滤
2. 对传入的参数要做类型校验，例如：整数数据，可以对数据进行整数强制转换
3. 保证格式化字符串的正确性，例如：int类型参数的拼接，对于参数要用%d，不能用%s

**错误做法：**
```python
print os.system("ls " + sys.argv[1])
```

**正确做法：**
```python
print os.listdir(sys.argv[1])
```

## 2. 异常行为

### 2.1 不要抑制或者忽略已检查异常

| 规则编号 | 规则名称 | 风险等级 | 说明 |
|---------|---------|---------|------|
| 2.1 | 不要抑制或者忽略已检查异常 | 高 | catch 块必须从异常中恢复或重新抛出适当异常，不能只是打印堆栈轨迹 |

**错误做法：**
```python
try:
    pass
except Exception:
    import traceback
    traceback.print_exc()
```

**正确做法：**
```python
validFlag = False
while not validFlag:
    try:
        validFlag = True
        pass
    except FileNotFoundException:
        import traceback
        traceback.print_exc()
```

### 2.2 禁止在异常中泄露敏感信息

| 规则编号 | 规则名称 | 风险等级 | 说明 |
|---------|---------|---------|------|
| 2.2 | 禁止在异常中泄露敏感信息 | 高 | 当异常会被传递到信任边界以外时，必须同时对敏感的异常消息和敏感的异常类型进行过滤 |

**正确做法：**
```python
turepath = os.path.realpath(sys.argv[1])
if turepath.startswith('/home/python/'):
    try:
        open(turepath)
    except IOError:
        print "Invalid file"
else:
    print "Invalid file"
```

### 2.3 方法发生异常时要恢复到之前的对象状态

| 规则编号 | 规则名称 | 风险等级 | 说明 |
|---------|---------|---------|------|
| 2.3 | 方法发生异常时要恢复到之前的对象状态 | 高 | 使用回滚、finally 子句、输入校验等方式维持状态一致性 |

**常用手段：**
1. 输入校验（如校验方法的调用参数）
2. 调整逻辑顺序，使可能发生异常的代码在对象被修改之前执行
3. 当业务操作失败时，进行回滚
4. 对一个临时的副本对象进行所需的操作，直到成功完成这些操作后，才把更新提交到原始的对象
5. 避免去修改对象状态

## 3. 文件操作

### 3.1 禁止使用mktemp创建临时文件

| 规则编号 | 规则名称 | 风险等级 | 说明 |
|---------|---------|---------|------|
| 3.1 | 禁止使用mktemp创建临时文件 | 高 | mktemp 返回的临时文件名中含有进程ID号，容易被猜测，存在符号链接攻击风险。Python2.3 版本之后该函数被废除，应使用 `mkstemp()` 或 `NamedTemporaryFile()` |

**错误做法：**
```python
import tempfile
tempfile = tempfile.mktemp()
```

**正确做法：**
```python
import tempfile
tempfile = tempfile.mkstemp()
```

### 3.2 临时文件使用完毕应及时删除

| 规则编号 | 规则名称 | 风险等级 | 说明 |
|---------|---------|---------|------|
| 3.2 | 临时文件使用完毕应及时删除 | 高 | 每一个程序在正常运行过程中都有责任确保删除已使用完毕的临时文件 |

**正确做法：**
```python
import tempfile
try:
    a = tempfile.NamedTemporaryFile(delete=True)
    a.write("abc")
finally:
    a.close()
```

### 3.3 在多用户系统中创建文件时指定合适的访问许可

| 规则编号 | 规则名称 | 风险等级 | 说明 |
|---------|---------|---------|------|
| 3.3 | 在多用户系统中创建文件时指定合适的访问许可 | 高 | 当一个程序在创建文件时没有对文件的访问许可做足够的限制，攻击者可能在程序修改此文件的访问权限之前对其进行读取或者修改 |

**正确做法：**
```python
import os
fd = os.open("tempnam.tmp", os.O_WRONLY, int("0600", 8))
myFileObject = os.fdopen(fd)
myFileObject.write("...")
myFileObject.close()
```

## 4. 序列化和反序列化

### 4.1 禁止对不可信来源的数据进行unpickle和load shelf

| 规则编号 | 规则名称 | 风险等级 | 说明 |
|---------|---------|---------|------|
| 4.1 | 禁止对不可信来源的数据进行unpickle和load shelf | 极高 | pickle 不提供安全性保证，反序列化不可信数据可导致任意代码执行。shelf 依赖 pickle，同样存在风险 |

### 4.2 将敏感对象发送出信任区域前进行签名并加密

| 规则编号 | 规则名称 | 风险等级 | 说明 |
|---------|---------|---------|------|
| 4.2 | 将敏感对象发送出信任区域前进行签名并加密 | 高 | 敏感数据传输过程中要防止窃取和恶意篡改，使用安全的加密算法加密传输对象 |

**适用场景：**
1. 序列化或传输敏感数据
2. 没有使用类似于SSL传输通道
3. 敏感数据需要长久保存

### 4.3 禁止序列化未加密的敏感数据

| 规则编号 | 规则名称 | 风险等级 | 说明 |
|---------|---------|---------|------|
| 4.3 | 禁止序列化未加密的敏感数据 | 高 | 序列化信息中不应该包括：密钥、数字证书、以及那些在序列化时引用敏感数据的类 |

**正确做法：**
```python
class GPSLocation:
    def __init__(self, x, y, id):
        self.x = x
        self.y = y
        self.id = id

    def __getstate__(self):
        state = dict(self.__dict__)
        del state['x']
        del state['y']
        return state
```

## 6. 其他

### 6.1 禁止在日志中保存口令、密钥等敏感信息

| 规则编号 | 规则名称 | 风险等级 | 说明 |
|---------|---------|---------|------|
| 6.1 | 禁止在日志中保存口令、密钥等敏感信息 | 高 | 口令包括明文口令和密文口令。若必须打印日志，用固定长度的星号代替敏感信息 |

### 6.2 禁止使用私有或者弱加密算法

| 规则编号 | 规则名称 | 风险等级 | 说明 |
|---------|---------|---------|------|
| 6.2 | 禁止使用私有或者弱加密算法 | 高 | 禁止使用 DES、MD5 等，应使用经过验证的、安全的、公开的加密算法 |

### 6.3 基于哈希算法的口令安全存储必须加入盐值（salt）

| 规则编号 | 规则名称 | 风险等级 | 说明 |
|---------|---------|---------|------|
| 6.3 | 基于哈希算法的口令安全存储必须加入盐值（salt） | 高 | 加盐后：密文 = 哈希算法（明文+盐值），增加口令破解难度 |

### 6.4 禁止将敏感信息硬编码在程序中

| 规则编号 | 规则名称 | 风险等级 | 说明 |
|---------|---------|---------|------|
| 6.4 | 禁止将敏感信息硬编码在程序中 | 极高 | 敏感信息可从 pyc 文件中获取，任何能够访问到 pyc 文件的人都可以获取这些敏感信息 |

**错误做法：**
```python
class Defpassword:
   defPassword = "123456"
```

### 6.5 使用安全随机数

| 规则编号 | 规则名称 | 风险等级 | 说明 |
|---------|---------|---------|------|
| 6.5 | 使用安全随机数 | 高 | `random` 模块产生伪随机数，不应用于安全用途。Linux/Unix 使用 `/dev/random`，Windows 使用 `SystemRandom` 类 |

**错误做法：**
```python
import random
sr = random.randint(0, 100)
```

**正确做法：**
```python
file = open("/dev/random", 'rb')
sr = file.read(10)
file.close()
```

### 6.6 使用SSLSocket代替Socket来进行安全数据交互

| 规则编号 | 规则名称 | 风险等级 | 说明 |
|---------|---------|---------|------|
| 6.6 | 使用SSLSocket代替Socket来进行安全数据交互 | 高 | SSL/TLS 提供完整性保护、认证和保密性 |

**SSLSocket 提供的保护：**
1. 完整性保护：SSL防止消息被主动窃取者篡改
2. 认证：在大多数模式下，SSL都对对端进行认证
3. 保密性（隐私保护）：SSL对客户端和服务器之间传输的数据进行加密

### 6.7 对子类继承的变量要做显式定义和赋初值

| 规则编号 | 规则名称 | 风险等级 | 说明 |
|---------|---------|---------|------|
| 6.7 | 对子类继承的变量要做显式定义和赋初值 | 中 | 子类没有定义的属性会引用基类的属性值，如果基类的属性值发生变化，子类引用的值也相应变化 |

**正确做法：**
```python
class B(A):
    x = 2
class C(A):
    x = -1
```

### 6.8 禁止通过注释的方式删除无用的功能代码

| 规则编号 | 规则名称 | 风险等级 | 说明 |
|---------|---------|---------|------|
| 6.8 | 禁止通过注释的方式删除无用的功能代码 | 高 | Python 运行时编译的特殊性，注释的代码可被轻易启用，必须彻底删除 |

### 6.9 慎用__del__方法（建议）

| 规则编号 | 规则名称 | 风险等级 | 说明 |
|---------|---------|---------|------|
| 6.9 | 慎用__del__方法 | 中 | `__del__` 可能引发异常，建议显式清理或使用 `atexit` 模块 |

**正确做法：**
```python
import atexit
connection = open_connection("deaddot.com")
   
def cleanup():
    print "Going away..."
    close_connection(connection)

atexit.register(cleanup)
```

## 危险函数清单

### 禁止使用的函数

| 函数 | 风险等级 | 说明 |
|------|---------|------|
| `eval()` | 极高 | 可执行任意 Python 代码 |
| `exec()` | 极高 | 可执行任意 Python 代码 |
| `input()` (Python2) | 极高 | 等同于 `eval(raw_input())` |
| `os.system()` | 极高 | 命令注入风险 |
| `os.popen()` | 极高 | 命令注入风险 |
| `pickle.loads()` | 极高 | 反序列化不可信数据可导致代码执行 |
| `pickle.load()` | 极高 | 同上 |
| `shelve.open()` | 高 | 依赖 pickle，同样存在风险 |
| `tempfile.mktemp()` | 高 | 已废弃，存在竞争条件 |
| `random.random()` | 中 | 伪随机数，不应用于安全用途 |

### 需要谨慎使用的函数

| 函数 | 风险等级 | 说明 |
|------|---------|------|
| `subprocess.call()` | 高 | 需避免拼接不可信输入 |
| `subprocess.Popen()` | 高 | 同上 |
| `open()` | 中 | 需验证路径合法性 |
| `os.path.abspath()` | 中 | 不解析符号链接，应使用 `realpath()` |
